# Product Specification: memo-everywhere

## 프로젝트 소개

**프로젝트명**: memo-everywhere

**한 줄 설명**: 모든 플랫폼에서 음성 입력, 자동 STT, 선택적 로그인, 양방향 동기화를 지원하는 크로스플랫폼 메모 애플리케이션

**관련 문서**: [[index]] | [[tech]] | [[structure]]

---

## 타겟 오디언스

- 멀티디바이스 사용자 (휴대폰, 태블릿, 데스크톱, 웹 동시 사용)
- 음성 입력을 선호하는 사용자 (운전 중, 손이 바쁠 때)
- 조직화된 메모 관리를 원하는 생산성 지향 사용자
- 클라우드 동기화를 필요로 하는 팀 협업 사용자

**주요 페르소나**:
- **퍼소나 A**: 바쁜 전문가 - 빠르게 아이디어 캡처, 음성 입력 활용
- **퍼소나 B**: 학생 - 강의 노트, 다중 태그로 과목별 정리
- **퍼소나 C**: 원격 작업자 - 모든 디바이스에서 접근, 팀 공유

## 핵심 기능

### 1. 텍스트 메모 작성 #feature/text
- 빠른 메모 입력 및 편집 기능
- 순수 텍스트 편집 (마크다운 렌더링 미지원)
- 온라인/오프라인 동작

### 2. 음성 기록 및 STT #feature/voice ✅
- 음성 녹음 후 자동 텍스트 변환 — **실제 구현 완료**
- 디바이스 네이티브 STT (ko-KR, en-US) + Whisper API 클라우드 STT
- 플랫폼별 코덱: iOS/macOS AAC, Android MP4, Web WAV
- 변환 실패 시 오디오 파일 보존 (에러 상태)
- 실기기(안드로이드) 검증 완료
- 관련: [[voice-processing]]

### 3. 선택적 로그인 및 계정 #feature/auth ✅
- **앱은 로그인 없이도 완전히 동작**: 로컬 메모, 로컬 STT
- **로그인 후 활성화**: 클라우드 동기화 + Whisper STT
- JWT 기반 인증 (24h 엑세스, 30d 리프레시)
- **세션 자동 복원**: 앱 재시작 시 로그인 상태 자동 복원 (재로그인 불필요)
- 홈 AppBar 계정 아이콘 → 로그인/회원가입

### 4. 양방향 클라우드 동기화 #feature/sync ✅
- **30초 주기 자동 폴링** (포어그라운드 시에만)
- Last-Write-Wins (LWW) 충돌 해결
- **영속 오프라인 큐** (Hive): 앱 강제종료 후 재시작해도 대기 작업 유실 없음
- **Access 토큰 자동 갱신**: 401 발생 시 refresh 토큰으로 자동 재발급 (끊김 없는 사용)
- 증분 동기 지원 (since 쿼리 + include_deleted)
- 소프트 삭제 지원
- 로그인 필수
- 관련: [[sync-strategy]]

### 5. 풍rich 한 검색 기능 #feature/search ✅
- **실제 구현 완료**: 제목/내용 키워드 전문 검색
- 대소문자 무시, 300ms 디바운스
- 음성 메모 전사본 검색 가능

### 6. 크로스플랫폼 지원 #feature/platform ✅
- **iOS**, **Android**, **macOS** ✅, **Web** ✅
- 플랫폼별 네이티브 권한 (마이크, Keychain)
- 반응형 레이아웃
- 관련: [[tech]]

### 7. 태그/폴더 조직화 #feature/organization (향후)
- 메모를 체계적으로 분류하고 관리
- 계층적 폴더 구조
- 복수 태그 지원
- 즐겨찾기 폴더

## 사용 사례

### 사용 사례 1: 이동 중 음성 메모 #usecase/mobile
**시나리오**: 운전 중에 생각난 아이디어를 음성으로 녹음
- 스마트폰에서 음성 녹음
- 자동으로 텍스트 변환 (#feature/voice)
- 태그로 프로젝트 분류 (#feature/organization)
- 나중에 데스크톱에서 정리 및 검색

### 사용 사례 2: 회의록 및 강의 노트 #usecase/educational
**시나리오**: 회의나 강의 내용을 음성으로 캡처
- 음성 녹음으로 회의/강의 내용 기록
- 자동 STT로 텍스트 변환
- 주제별 태그로 분류 (예: "프로젝트명:A", "강의:수학")
- 전문 검색으로 빠르게 찾기

### 사용 사례 3: 멀티디바이스 메모 접근 #usecase/sync
**시나리오**: 여러 디바이스에서 메모에 접근하고 수정
- 아침: 스마트폰에서 메모 시작
- 점심: 태블릿에서 계속 추가
- 저녁: 데스크톱에서 정리 및 내보내기
- 모든 변경사항이 실시간 동기화

### 사용 사례 4: 프로젝트별 메모 관리 #usecase/organization
**시나리오**: 프로젝트별 메모를 체계적으로 관리
- 각 프로젝트별 폴더 생성
- 관련 메모를 폴더에 저장
- 폴더 간 빠른 전환
- 팀 멤버와 폴더 공유 (향후 기능)

## 구현 제외 사항 (Non-Goals)

### ❌ 텍스트 형식 지정 (Rich Text)
- 마크다운 또는 WYSIWYG 에디터 미지원
- 순수 텍스트만 지원
- **이유**: 동기화 복잡도 증가, 모바일 UX 악화

### ❌ AI 요약 및 생성
- 메모 자동 요약 미지원
- AI 기반 작성 보조 기능 없음
- **이유**: 외부 API 의존성 증가, 비용 관리 어려움

### ❌ 이미지/파일 첨부
- 메모에 사진 첨부 미지원
- 파일 첨부 미지원
- 텍스트와 음성만 지원
- **이유**: 스토리지 비용, 동기화 복잡도

### ❌ 협업 편집
- 실시간 다중 사용자 편집 미지원
- 동기화는 개인 클라우드 계정 기준
- **이유**: 충돌 해결 복잡도, 보안 이슈

### ❌ 메모 번역
- 다국어 번역 기능 미지원
- **이유**: 외부 API 의존성, 비용

### ❌ 알림/리마인더
- 메모 기반 알림 기능 미지원
- To-Do 리스트 관리 미지원
- **이유**: 제품 범위 외, 별도 앱으로 구현 권장
